# SEO Report — LLM Fit Check (llmfitcheck.com)

*Researched and applied June 2026. Claims verified against live Google Search Central docs,
web.dev, Bing docs, and primary studies (sources at the bottom).*

## TL;DR

Google indexes client-rendered React fine in 2026 (evergreen Chromium, median render delay
~10s). **The engines that don't are the ones that matter most for this tool**: Bing's JS
rendering is unreliable (and feeds DuckDuckGo), and the AI search crawlers — GPTBot,
OAI-SearchBot, ClaudeBot, PerplexityBot — **execute no JavaScript at all**. Queries like
"how much VRAM do I need for a 70B model" are exactly what people ask ChatGPT and
Perplexity, so the single highest-impact decision was to ship the explainer content as
**real static HTML in `index.html`**, not as React-rendered DOM.

The second finding that shapes strategy: ranking for a brand-new domain is link-bound.
On-page work is table stakes; **Show HN (followed links!), one genuine r/LocalLLaMA post,
and awesome-list PRs** are what actually move the needle.

## What was applied

### High impact
| Item | Implementation |
|---|---|
| **Static, crawlable content** | Explainer ("How LLM Fit Check works"), a **quotable VRAM table** (7B→70B at Q4_K_M/FP16 — the Princeton GEO study found statistics/direct numbers boost generative-engine citation 30–40%), and a 6-question FAQ now live as plain HTML in `index.html`. Visible to users below the app; readable by every crawler without JS. The React tree no longer duplicates them. |
| **Title/description** | `LLM Fit Check — VRAM & RAM Calculator for Local LLMs` (~55 chars, keyword-leading) + ~150-char meta description written for click-through (Google rewrites ~⅔ of descriptions; it's a snippet candidate, not a ranking factor). |
| **Canonical + single origin** | Self-referencing canonical to `https://llmfitcheck.com/`; Caddy 301s `www.` and HTTP to the apex over HTTPS. |
| **Structured data** | Static JSON-LD `@graph`: `WebApplication` (DeveloperApplication, `isAccessibleForFree`, free offer) + `WebSite` (controls the site name in results). No fabricated ratings — the Software-App *rich result* needs reviews we don't have; the markup is entity description for Google/Bing/AI engines. |

### Medium impact
| Item | Implementation |
|---|---|
| Social cards | OG (type/site_name/title/description/url) + 1200×630 `og-image.png` with width/height/alt, `twitter:card summary_large_image`. One image serves Facebook/LinkedIn/Slack/Discord/X. |
| Favicons | `favicon.svg` + `favicon.ico` 32px fallback (SVG unsupported on iOS ≤ 18), 180px `apple-touch-icon`, `manifest.webmanifest` with 192/512 icons. Generated by `scripts/make_assets.py`. |
| Performance | Self-hosted variable fonts (woff2, preloaded, `font-display: swap` — the Google Fonts CDN cache-sharing benefit died with Chrome cache partitioning); gzip app-side + zstd/gzip at Caddy; `Cache-Control: immutable` for hashed assets, `no-cache` for index.html. Bundle is ~62KB gzipped — far under any 2026 budget, so no code splitting. |
| Semantics | Real `<h1>` (brand), `h2`/`h3` hierarchy in the static section, `lang="en"`, theme-color. |
| robots.txt + sitemap.xml | Allow-all + sitemap pointer; single-URL sitemap with `lastmod` only (`changefreq`/`priority` are officially ignored). Tiny sites don't *need* a sitemap, but it's free discovery insurance for a zero-backlink domain and unlocks GSC reports. |

### Deliberately skipped (and why)
- **SSR / prerender service** — overkill for one URL; Google's dynamic rendering is officially deprecated; static HTML content achieves the same goal for non-JS crawlers.
- **`noscript` content** — Google ignores noscript body text; replaced with visible static HTML (removed ours).
- **`llms.txt`** — verified: no AI engine consumes it (Google on record comparing it to the keywords meta tag).
- **FAQPage / HowTo / SearchAction JSON-LD** — all dead: Google discontinued FAQ rich results entirely as of May 7 2026 (the 2023 gov/health carve-out is gone). The *visible* FAQ text is what matters now.
- **meta keywords, `robots: index,follow`** — no-ops.
- **Code splitting, zstd precompression, CWV micro-tuning** — bundle and metrics already pass with huge headroom; CWV is a tie-breaker, not a lever (Google: relevance wins "even if the page experience is sub-par").

## Launch playbook (the part that actually moves rankings)

1. **Google Search Console**: add a *Domain property* for `llmfitcheck.com` (DNS TXT verification — covers apex/www/http/https), submit `sitemap.xml`, then URL Inspection → **Request Indexing** for `/` (same-day-ish first index).
2. **Bing Webmaster Tools**: one-click import from GSC. Feeds DuckDuckGo, and ChatGPT retrieval still leans on Bing. Watch the new **"AI Performance" report** (Feb 2026) — the only first-party ChatGPT/Copilot citation telemetry.
3. **Show HN** — story links are followed (verified June 2026): real PageRank plus 100+ persistent links from HN mirrors. Best single launch action.
4. **r/LocalLLaMA** — one genuine, detailed post. Reddit is licensed into both Google and OpenAI training/search; cited threads are typically small Q&A posts, not viral hits. Don't astroturf.
5. **Awesome-list PRs** (awesome-llm, awesome-local-llm etc.) + link from the GitHub README.
6. **Keep the model table fresh** — the page says data refreshes live; revisit the static table when new model families land (freshness is real for ChatGPT citation).
7. Validate the social card once live: https://www.opengraph.xyz.
